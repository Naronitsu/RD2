\begin{abstract}
This paper investigates minimal, resource-bounded runtime verification for safety-relevant automotive ECU telemetry. A server-side replay pipeline was implemented using LARVA monitors and benchmarked under two regimes: baseline (clean trace) and abnormal (injected violations), while toggling runtime verification on/off on the same endpoint. The evaluation assumes telemetry records are forwarded from an IoT/ECU endpoint to a server-side verifier. Four properties were evaluated: voltage out-of-range under running RPM, coolant over-temperature, throttle spike, and load--MAP mismatch. Across 10 measured runs per mode, baseline overhead was 8.34\% (113.0 ms RV on vs 104.3 ms RV off), while abnormal overhead rose to 3951.94\% (4274.8 ms vs 105.5 ms), showing strong state-dependent monitoring cost. Detection behaviour was coherent: injected violations produced high RV-on flag counts while RV-off flags remained zero. Threshold tuning reduced baseline false positives for load--MAP mismatch without removing abnormal detections. The findings support the feasibility of lightweight monitoring in nominal conditions, but also show that violation-heavy paths require optimisation before deployment in strict real-time ECU environments.
\end{abstract}
